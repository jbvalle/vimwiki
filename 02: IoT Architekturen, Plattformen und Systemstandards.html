IoT Architekturen, Plattformen und Systemstandards

Systemanforderung an IoT Architekturen

  • Skalierung:
    • Anpassung eines IT Netzwerkes auf ein Iot ist weit großer
  • Security:
    • Bei drahtlose Kommunicationsprotokollen sind die daten der Außenwelt ausgesetzt, muss dementsprechend abgesichert werden
    • Sobald daten auch angekommen sind, müssen zugriffsrechte festgelgegt werden
  • Limitierung:
    • zu berücksichtigen:
      • Gegrenzte Leistung
      • CPU
      • Speicher
      • Übertragungsrate
  • Datenolumen:
    • Handhabung big data -> Engpässe handhaben
  • Altgeräte:
    • Handling von incopatibility aufgrund fehlendem support

Erhöhung der Übertragungsweite

  • Eingrenzung der Frequenz mit der selben leistung ermöglicht größere Signalausbreitung

Arten von IOT Architekturen

IOTWF

  • Ähnlich wie OSI Model
  • 7 schichte Mode:
    • 1. Physical Devices:
      • Thing in IOT
    • 2. Connectivity:
      • Processing units for connection establishment, switch, rounter, etc.
    • 3. Edge Computing:
    • 4. Data Accumulatio:
      • Datenspeicherungo
    • 5. Data Abstraction:
      • Handling unterschiedlicher Protokolle und Datenformate und zusammenfügung zu einer einheitlichen Abstraktion
    • 6. Application:
    • 7. Collaboration and processes:
      • Die ansammlung der daten werden als services weitergegeben um nützlich zu sein

Netzwerk Grundlagen

Topologien

  • Verbindungsart von knoten
  • Unterschieden wird zwischen phykalischen Topologien, hardware Verbindungsart, und logische Topologie, tatsächlicher Signalfluss
  • Beispiel:
    • Sternstruktur:
      • Alle knoten sind mit einem zentralen knoten vebunden
    • Baumstruktur:
      • Knoten ähnlich wie zweige eines baumes verbunden
    • Maschennetz:
      • Jeder knoten ist mit einem anderen oder mehreren knote verbunden
      • Vorteil:
        • Bei ausfall, ist es oft möglich über umleitungen zum ziel zu kommen

Rounting

  • Festlegen von Datenpfaden
  • Methoden:
    • Staische Rounting: Manuelles Konfigurieren von Datenpfaden
    • Adaptives/Dynamisches Rounting: Bei großen Topologien, bzw. dynamischen Topologien, können durch routing tables, der Pfad automatisch angepasst werden:
      • Rounting Table:
        • Merkt sich datenpfade ähnliche wie eine Map und kann anhand dieser information neue rounting pfade generieren

Informationsaustauscharten

  • Simplex:
    • Communication nur in eine Richtung
  • Halbduplex:
    • Abwechselnd entweder in eine oder die andere Richtung
  • VollDuplex:
    • Kann gleichzeitig in beide richtungen kommunizieren

Weitere Begriffe

  • Telemetrie:
    • Messen, steuern, und regeln von der distanz
  • Telemonioring:
    • Messen und übewachen von der distanz
  • Telesurveillance:
    • Aufbuen eines strukturierten netzwerkes zur fernüberwachung -> beinhaltet mehrere subsysteme
