Assembly Language

In the earlier years of microcontrollers not many years ago, programming was mostly done by using Assembly Language. Today in a few cases its actually still worthwhile programming in assembly, like ISR for better efficiency. Few bit operations cannot be properly represented in C, hence assembly is necessary like bitwise rotations or binary-coded-decimal arithmetic and detailed register manipulations.

One of the downside of Assembly is that it is not very portable as some assembly directives might vary depending on vendors' assembler for the same device. However in some case it's still useful to learn assembly, especially for debugging, detailed interaction between software and hardware and peripherals.

Assembly Layout

A typical assembly line has four parts:

  • label: it starts in the first column and may be followed with a semi colon
  • operation: this could either be an instruction which is then translated into binary or a directive which directly controls the assembler
  • operand: data which needed for the instruction if necessary
  • comment: at the end of the line, start with ;

Number representation

Earlier hex was used, newer assembler usually use decimal. For Hex representation there are multiple possible notion:

  • $A5
  • h'A5'
  • 0A5h -> prefered by MSP430

MSP430 Assembly Guidlines

  • No line should exceed 80 chars
  • Use macros provided by msp430 headers
  • labels start at column 1 and are 10 chars or fewer
  • Instructions or directives start at column 13
  • instructions are lower case and DIRECTIVES are UPPERCASE
  • Operands start at column 21
  • Comments start at column 45 and the first char is capitalized -> for multiline comments this only counts for the first line
