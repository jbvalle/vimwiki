Replication

  • Scaling by replication is a solution the the latency problem caused by geographically distant server to client routes
  • By create multiple instances of a resource the latency can be hidden from the client

Downsides

  • However with replication comes another problem, that is inconsistency. If many replications of an instance exists, changes have to propagated throughout the system to stay up to date
  • Furthermore, in case multiple different changes of replications have be performed, this has to be tackled as well
  • The Developer has to wheigh out the here, to what extent inconsistency is tolarable
  • An example is caching, in this case the client decides if he wants to use that option, in that case a replication is made of that resource. However if changes are made in may not occure immidiately. If it were for stock market exchanges, updates however must be immidiate, in which is inconstency is not as tolerable
