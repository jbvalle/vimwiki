Gnu Assembly Directives

Assembly Directives

Directives for selecting the current Section

Selections are used to devide the system memory into independent groups of segments which are referenced by pointers located in the segment register. Each of these Segments can be subdivided into further sub segment. Each Section contain specific types of data, .data (for static memory) and .bss (these are initialized with zero), .text for machine instructions and stack for parameters, local variables automatically allocated variables.

  • .data usually used for global variables and constants which have labels
  • .text used for executables as well as constants
  • .bss (block started by symbol) used for defining data storage areas which are initialized with zero, usually used for global variables. This section doesnt consume memory space, as it just request the loader to reserve memory space when the program is loaded to memory.
  • .section name, a custom section can be created used using this directive. In order for custom sections to be recognized by the linker, the linker first have to be made aware of them.

Allocating memory for variables and constants

  • .byte expr: this directive expects zero or more expr and allocates byte long variables
  • .hword expr and .short expr : 16 bit long expressions
  • .word expr and .long expr : 4 Byte long expressions -> 32 Bit
  • .ascii "string" : Assembles each string into consectuve addresses -> no trailing null char
  • .asciz "string" and .string "string" : Similar to .ascii but appends a null char
  • .float float_num and .single float_num : 4 Byte long single precision
  • .double float_num : 8 Byte double precision number

Filling and aligning

  • .align expr1, expr2, expr3 :
    • expr1: specifies number of low order zero bits of location counter after advancement
    • expr2: specifies value to be stored in the padding bytes (this as well as the comma can be ommited)
    • expr3: specifies the maximum number of bytes that are allowed to be filled with (just as expr2 this is optional)
  • .balign [lw] expr1, expr2, expr3 :
    • expr1 : defines the byte multiple: ie .balign 2 fill as many bytes as needed until the location counter is a multiple of 2
    • expr2 : defines the fill value that should be used
    • expr3 : defines the max number of bytes that is to be skipped
    • .balignl and .balignw are variants :
      • .balignl treats the fill pattern (expr2) as 2 bytes meaning expr2 will be filled in every 2 bytes
      • .balignw treats the fill pattern (expr2) as 4 bytes meaning expr2 will be filled in every 4 bytes
  • .skip byte_size, fill and .space byte_size, fill :
    • this is used for specifying large number of bytes and initializing them with fill. This is often used in .bss section
