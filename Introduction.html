1. Introduction

Reasons for learning Assembly

  • handle boot process of computer
  • interrrupt handler code
  • low level locking code for multi threading
  • code for machines without a compiler
  • for optimizing code beyond the capability of a compiler
  • coding on computers with limited memory
  • code that requires low level access on architectural and processor features

Binary Representations

  • Complement (ie, 0 ... 255)
  • Sign-magnitude (ie, 0 .. 127,-0 ... -127)
  • excess-N (ie, -127 ... 0 ... 128)
  • Two's Complement (ie, 0 ... 127, -128 ... -1)

Char Representations

  • A standarized version is ASCII - American Standard Code for Information Interchange:
    • As ASCII was published in the 1960s telephone wires were used for transmission
    • In order to inhance performance, due to the very slow data transmission, as the data had to be converted into audo signals which is then decoded by an acoustic modem, the ASCII Code had been defined as 7 Bit code, this led to an approximately 10% better data transmission performance.
    • However in order to provide information on how the character should be formatted, the total 8 Bits have been used, adding extra 33 Characters serving as character formatting Information.

Memory Layout of Executables

  • little endian systems:
    • LSB in lowest memory address
    • MSP in highest memory address
  • Big endian systems:
    • LSB in highest memory address
    • MSP in lowest memory address
  • Typical Layout:
    Memory Address Memory Type
    ---------------- ------------
    0xFFFF ....
    .... Stack
    .... Currently unused
    .... Heap
    .... Data
    .... Text
    0x0000 ....
  • Text:
    • Program Code containing executable instructions
    • Size Depends on program size and pre declared data storage by the programmer
  • Data:
    • Contains statically allocated variables, which when initialized have fixed addresses which are unchangable onces the program is running
    • Size Depends on program size and pre declared data storage by the programmer
  • Heap:
    • Contains dynamically allocated variables -> these are allocated using malloc/new/or-any-other-quivalent
    • The address if a dyn. Variable is always stored in another variable -> pointer -> which could be an automatic variable in stack or a statically allocated variable in the data section
  • Stack:
    • Contains used parameters for function calls, return addresses and local variables (automaticall allocated variable, once such a variable is allocated the stack pointer is adjusted down to make space for a new variable. These Variables are always computed as an offset of the stackpointer)
