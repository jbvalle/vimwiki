Von Neuman and Harvard architecture

Von Neumann

  • Every component is connected by a connected centralized bus
  • The cpu is controlled by the content of the memory

Diagram Von Neuman

CPU + | +------------+--------------+ Memory | | +--------------+ Peripherals: IN & OUT

Harvard Architecture

  • The Program Code Memory and Data Memory are seperated and both interface the CPU by different busses

Diagram Harvard

CPU +---------- Intruction memory (for program code) + | +-------+----- Data Memory (for variables) | +----- Peripherals: IN & OUT

Main Differences

Von Neumann Harvard
Single bus is simpler design multiplex is more complex
Common memory for data and instructions seperate memory space for data and instructions
Simultaneous multiple memory fetches are not allowed Allows 2 simultaneous memory fetches
