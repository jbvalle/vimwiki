Endianess

  • for setting little or big Endianess set Bit 9 of the Program Status Register CPSR

Little Endianess

Byte 3 : Higher Address Byte 2 Byte 1 Byte 0 : Lower Address

Big Endianess

Byte 3 : Lower Address Byte 2 Byte 1 Byte 0 : Higher Address

ARM Registers

R0 - R6 General Purpose
R7 Holds Syscall Number
R8 - R10 General Purpose
R11 (Alias: FP) Frame Pointer

Special Purpose Register

R12 (Alias: IP) IIntra Procedural Call
R13 (Alias: SP) Stack Pointer
R14 (Alias: LR) Link Register
R15 (Alias: PC) Program Counter
CPSR Current Program Status Register

R13 - Stack Pointer

  • Points to the top of the stack
  • Once a function is called, we return back to the location last pointed to by the SP
  • Another Purpose is for Space Allocation -> for allocating a 32 Bit value we subtract 4 from the stack pointer

R14 - Link Register

  • Once a function call is initiated, the memory space in which the next instruction is stored is loaded into the Link register
  • This helps us to return to the location the function call has first initiated an instruction
  • This helps to return to the parent function which has initiated further child functions

R15 - Program Counter PC

  • The PC is automatically incremented by the size of the instruction executed.
  • In ARM Mode its its 4 Bytes and in thumb mode its 2 Bytes
  • Once a branch instruction is executed the PC hold the destination address that is beeing branched to
  • During execution the PC stores the address of the current instrution plus 8 in ARM mode and plus 4 in thumb mode

Overview CPSR

N 1 if result of the instruction is a negative number
Z 1 if result is a 0
C 1 if result of an addition needs 33rd bit -> carry, if the result of a subtraction is positive or zero, as the result of an inline barrel shifter operation in a move or log. instruction
V 1 if result cant be represented in 32 bit twos compliment , if result is >= 2^31
E 0 for little endian , 1 for big endian
T 1 if in Thumb state and 0 for ARM state
M Specifies the privilege mode (USR , SVC , etc.)
J Third execution state that allows some ARM Processors to execute Java Byte Code in Hardware
