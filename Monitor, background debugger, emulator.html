Access to MCU for programming and debugging

In order to communicate with the MCU for programming and debbuging with need a two way communication protocol to send data for programming and recieve data for debugging.

Three common interfaces for MCUs are:

  • Bootstrap bootloader (BSL): Serial interface. The BSL needs only a level shifter for connection with RS232 ports. However this interface could only be used for programming and not debugging, as debugging was mostly done by serial monitors. The BSL was permanently stored in masked ROM and was executed instead of the user program, when specific debugging pins of the MCU were signaled and the MCU was reset. The big disadvantage of this interface is that it is very intrucive as the user's program had to com halt completely in order for the debugger to take control.
  • Conventional JTAG (four wires)
  • Spy-Bi-Wire JTAG (two Wire)
