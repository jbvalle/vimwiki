Risc Design Philosophy

ARM Company

  • Mainly a design firm - do not produce hardware
  • Sell the licenses to brands

RISC

  • Reduces instruction set computer used by ARM Processors
  • Contrary CISC is used by x86 Processors
  • Instructions are executed in parallel by pipelines
  • Have large general Purpose register set to be more flexible
  • Processors operates on data held in registers by load-store

ARM - Design Philosophy

  • ARM Cores are design for less power consumption
  • ARM is a compromis of RISC - High Code density - better Power Consumption

Fundamental Difference CISC and RISC

  • RISC:
    • RISC Design Philosophy is simplicity over complexity
    • Because of that, greater emphasis have been put unto the compiler which handles the simple instruction set and figures out the complex function behind it -> it then produces the simplified excutable
  • CISC:
    • In CISC the greater emphasis has been put unto the CISC Architecture design which is responsible for understanding the the complex instructions and executes the binary

Further Differences

  • Endianess:
    • Risc: Little Endian, big endian
    • CISC: little endian
