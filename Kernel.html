Kernel

The general path to exposing a hardware functionality by software is as follows.

  1. (Application Ring) User Process:
    1. this processes invokes function calls defined in LIBC
    2. This function call is redirected to LIBC
  2. (Shell Ring) LIBC:
    1. The function call which is defined in LIBC is executed
    2. The function call invokes the necessary systemcalls which are redirected to the kernel ring
  3. (Kernel Ring) Kernel Process:
    1. The Kernel Process executed the systemcalls programs with the highest priviliges making it possible to access the whole memory space and all devices

The Properties of a kernel

  • The kernel process is the first thing that is loaded and executed
  • The is only one kernel process, but multiple user processes are possible
  • The Use Process invokes the system calls and waits until the kernel process handles and executes all the system calls

Tasks of a kernel

  • Process Management:
    • User Processes are created by the kernel via system calls
    • The Kernel is responsible for allocating the needed memory, and load the needed instructions
  • Inter Process Communication (IPC):
    • The kernel is responsible to enable communicatio between processes though shared memories, pipes and unix domain sockets
  • Scheduling:
    • Scheduling is the action handled by a task to coordinate different processes and base them on priority and importance and handle multitasking.
  • Memory Management:
    • The Kernel is responsible for allocating and freeing memory needed by processes as well as all memory related tasks.
  • System StartUp:
    • After the kernel image is loaded into main memory, its job is to initialize the userspace, by creating the first user processes with Process ID 1 and initializing hardware.
  • Device Management:
    • The kernel is also responsible for handling real and virtual hardware by handling the memory mapped device files used by user processes for communication
