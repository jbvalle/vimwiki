distributed computing systems

This type of DS System in essence distributes the computation usually done buy only one server. This comes in 3 types:

Cluster Computing

  • consists of a collection of workstatios or pc connected by means of high speed LAN local network, usually running the same hardware and operating system
  • Such a system usually is administered by a single master node, managing all other nodes in the cluster
  • The middleware in this case actually run on the master node, while the other nodes are equipped with normal OS and middleware function to enable communication, storage, fault tolerance etc.
  • An Example of such a system is the Linux based MOSIX System. Its high transparency is achieved by applying dynamic preemptive process migration, making it possible to start a process in one node, but processing it at another node for most efficient use of resources

Grid Computing

  • Often constructed of enteties of computer systems, in which each system is asigned its own administrative purposes, which can all be different in software and hardware

Cloud computing

  • This type is an outsourced version of entire infrastructures and services to offer compute-intensive applications
