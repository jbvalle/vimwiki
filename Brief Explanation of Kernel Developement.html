2.1 What is Memory

Types

  • RAM: its the computers main memory with r+w accessability
  • ROM: Is memory that can only be read

RAM

  • Read and writable memory
  • Used only for temporary storage
  • Information is lost when power lost

ROM

  • Permanent storage, saves information even after power down
  • Can not be written to by normal means
  • Generally used for storing permanent memory such as program for embedded devices
  • Example : BIOS

Memory Structure

  • Memory is generally stored linearly

2.2 Boot Process

  • Executed from ROM -> using BIOS CHIP
  • BIOS loads the bootloader into address 0x7C00 -> The bootload loads the kernel

Bootloader

  • Its main task is to load the kernel of an OS
  • Generally manufacturers make their own bootloader, ie. Linus : grub
  • Generally small footprint
  • When the Computer first boots, it enters a compatibility mode called real mode
  • The Bootloader accesses functions within the BIOS to perform tasks
  • Main Task:
    1. Load kernel into memory
    2. Switch processor into 32Bit protected mode
    3. Which in turn enables processor to execute our kernel

Real Mode

  • In Real Mode, we only have access to 1MB of Memory and can only operate 16Bit code -> Thats why the bootloader usually is of a smaller footprint
  • The purpose of the bootloader is to then switch the processor mode to protected mode which then gives us access to 4GB of memory and lets us operate 32 Bit Commands

BIOS

  • After the computer boots up, the CPU wil execute instructions directly from the BIOS's ROM
  • The BIOS will load itself to RAM memory
  • responsible to init essential hardware
  • The BIOS will look for a bootloader to boot by searching in all storage mediums for a boot signature "0x55AA" -> Usually the 511th and 512th bytes contains this signature.
  • If the BIOS finds the singnature it in turn loads that memory sector into absoute address 0x7C00 and subsequently run from that address exercuting the bootloader by instructing the process to jump into that addres
  • The Bootloader is then executed

BIOS Mini Kernel

  • The BIOS is a kernel in itself
  • It contains routines to assist out bootloader in booting the kernel
  • The BIOS is 16 Bit Code which means only 16 Bit code can execute the BIOS
  • BIOS Routines are generic and standard
